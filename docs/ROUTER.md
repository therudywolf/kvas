# Управление роутером

## Установка пакета

1. Соберите .ipk (см. [BUILD.md](BUILD.md)) или возьмите готовый из `output/`.
2. Скопируйте .ipk на роутер, например в `/opt/tmp/`:
   ```bash
   scp output/frt_*.ipk root@<IP-роутера>:/opt/tmp/
   ```
3. На роутере установите пакет:
   ```bash
   opkg install /opt/tmp/frt_*.ipk
   ```
4. Выполните первичную настройку:
   ```bash
   frt setup
   ```

## Требования на роутере

- Роутер с [Entware](https://github.com/Entware/Entware) (например, Keenetic).
- Зависимости пакета: **jq**, **curl**, **dnsmasq-full**, **ipset**, **stubby** (DoT), **shadowsocks-libev** и др. — устанавливаются через `opkg` при установке FRT (см. Makefile).

## Команда frt

Единая точка входа — команда **frt** (устанавливается в `/opt/bin/frt`).

### Основные подкоманды

| Команда | Описание |
|---------|----------|
| **frt setup** | Первичная настройка после установки. |
| **frt show / list** | Показать защищённый список хостов. |
| **frt search / find \<домен\>** | Поиск домена в списке. |
| **frt add / new \<хост\>** | Добавить хост в список. |
| **frt del / rm \<хост\>** | Удалить хост из списка. |
| **frt import \<файл\>** | Импорт хостов из файла. |
| **frt export \<файл\>** | Экспорт списка в файл. |
| **frt purge / clear** | Очистить список (есть подтверждение). |

### VPN и DNS

| Команда | Описание |
|---------|----------|
| **frt vpn** | Статус VPN. |
| **frt vpn set** | Выбор VPN-интерфейса. |
| **frt vpn reset** | Обновить правила iptables. |
| **frt ssr** | Shadowsocks: статус и настройки. |
| **frt dns** | Текущий DNS. |
| **frt dns crypt on \| off \| status** | DoT (stubby): вкл/выкл/статус. |
| **frt dns test \<домен\>** | Тест DNS. |

### Теги, отладка, управление пакетом

| Команда | Описание |
|---------|----------|
| **frt tags list / add / del / update** | Теги доменов (закваски). |
| **frt test** | Проверка служб. |
| **frt debug** | Отладочная информация. |
| **frt log [N]** | Последние события пакета. |
| **frt version / ver** | Версия. |
| **frt upgrade / update** | Проверка и обновление пакета. |
| **frt pause / unpause** | Приостановка / возобновление работы. |
| **frt uninstall [full]** | Удаление пакета. |

Полный список и примеры: на роутере **frt help** или файл `opt/etc/conf/frt.help` в репозитории.

## Структура на роутере после установки

| Путь | Назначение |
|------|------------|
| **/opt/bin/frt** | Главная команда (симлинк или скрипт). |
| **/opt/apps/frt/** | Файлы пакета (скрипты, конфиги, ndm). |
| **/opt/etc/frt.conf** | Основной конфиг FRT. |
| **/opt/etc/frt.list** | Защищённый список доменов/IP. |
| **/opt/etc/dnsmasq.d/frt.dnsmasq** | Правила ipset для dnsmasq (генерируются из frt.list). |
| **/opt/etc/init.d/S96frt** | Init-скрипт. |
| **/opt/etc/ndm/** | Хуки ndm (100-frt-vpn, 100-dns-local, ifstatechanged.d, …). |

Цепочка DNS: клиент → dnsmasq (порт из конфига) → stubby (DoT) → upstream. AdGuard, Adblock, DNSCrypt в пакете не используются.

## Масштабирование (100k+ доменов)

- Список обрабатывается потоково; добавление — append в `frt.list`, удаление — однопроходное переписывание.
- В dnsmasq рекомендуется увеличить `dns-forward-max` (например до 1024).
- Подробнее: раздел «Масштабирование» в [README](../README.md).
