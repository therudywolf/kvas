# Тесты FRT

## Назначение

Тесты проверяют логику работы со списком и генерацию конфигов **без роутера**: парсинг `frt.list`, генерация dnsmasq, добавление/удаление хостов, граничные случаи. Запуск в среде с `sh` (Linux, Git Bash, WSL).

## Запуск

**Все тесты (точка входа):**
```bash
sh tests/run_tests.sh
```

Из корня репозитория:
```bash
./tests/run_tests.sh
```

Возвращаемый код: 0 — все тесты прошли, 1 — хотя бы один упал.

## Состав тестов

| Скрипт | Что проверяет |
|--------|----------------|
| **list_parsing.sh** | Парсинг списка: комментарии, пустые строки, IP, домены. |
| **dnsmasq_generation.sh** | Генерация строк ipset для dnsmasq из тестового списка. |
| **host_add_del.sh** | Добавление и удаление хостов во временном списке. |
| **error_cases.sh** | Пустой список, удаление несуществующей записи, дубликаты, спецсимволы, отсутствующий файл списка. |

Тесты используют временные файлы (см. `tests/env.sh`); реальный `/opt` и рабочие конфиги не трогают.

## Окружение

Переменные задаются в **tests/env.sh**:

- `TESTS_DIR` — каталог тестов.
- `FRT_LIST_FILE_TMP`, `FRT_DNSMASQ_TMP` — временные файлы списка и dnsmasq.
- `FRT_SCRIPT_DNSMASQ`, `FRT_SCRIPT_IPSET` — пути к скриптам в `opt/bin/main/`.
- `PATH` — дополняется каталогом `opt/bin`.

При необходимости можно переопределить переменные снаружи.

## Интеграция в CI

Полный цикл «сборка + тесты + валидация .ipk» выполняет скрипт **scripts/ci.sh** (локально или в Docker). Он вызывает `tests/run_tests.sh` после сборки; при падении тестов сборка считается неуспешной.

Пример:
```bash
./scripts/ci.sh
# или
docker-compose run --rm ci
```
